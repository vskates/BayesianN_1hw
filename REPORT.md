# Байес-оптимальный морской бой
## 2. Формализация байесовского хода

Пусть:
- `B` — расстановка кораблей;
- `S_t` —  информация к моменту `t`
- `c` — клетка, по которой можно выстрелить.

Пусть `q_t(c) = P(c занята кораблём | S_t)`.

Локально байес-оптимальный ход равен

`c_t^* = argmax_c q_t(c)`.

### Что считается в коде

В коде используется классическая **occurrence matrix / probability density**
аппроксимация:

1. Для каждого ещё не потопленного корабля генерируются все допустимые
   размещения, согласованные с текущими наблюдениями.
2. Для каждой клетки считается, в скольких допустимых размещениях она встречается.
3. Клетка с максимальным счётом выбирается как следующий ход.

Если на поле уже есть попадания по ещё не потопленному кораблю, то в расчёт
берутся только размещения, которые действительно покрывают эти попадания.

Это и есть реализация байесовского выбора по текущему постериорному распределению
над положениями кораблей, хотя строго глобально-оптимальная стратегия для всей
игры была бы уже POMDP-задачей и заметно сложнее.

## 3. Визуализация поля и распределения вероятностей

Скрипт `run_analysis.py` строит SVG-файлы:

- [outputs/example_heatmap.svg](outputs/example_heatmap.svg) — пример доски и
  тепловой карты вероятностей после появления первой информации;
- [outputs/shot_distribution.svg](outputs/shot_distribution.svg) — сравнение
  распределений числа ходов для разных стратегий;
- [outputs/summary.json](outputs/summary.json) — сырые итоговые числа эксперимента.

На `example_heatmap.svg` показаны:

- истинная скрытая доска;
- уже сделанные выстрелы;
- значения тепловой карты;
- рамкой выделен байес-оптимальный следующий ход.

## 4. Какие стратегии сравнивались

В эксперименте использованы две стратегии.

### 4.1. Random

Игрок просто выбирает случайную ещё не открытую клетку.

### 4.2. Bayes

Это основная стратегия проекта:

- пересчитывается occurrence matrix;
- выбирается клетка с максимальной апостериорной вероятностью попадания.

## 5. Эксперимент

### Настройка

- сгенерировано `80` случайных скрытых досок;
- все стратегии стреляют по одному и тому же набору досок;
- генератор случайных чисел зафиксирован seed-ами для воспроизводимости;
- метрика — общее число выстрелов до полного уничтожения флота.

### Результаты

По файлу `outputs/summary.json` получаем:

| Стратегия | Среднее | Медиана | Std | Min | Max |
| --- | ---: | ---: | ---: | ---: | ---: |
| Bayes | 55.21 | 56 | 7.00 | 36 | 76 |
| Random | 75.49 | 76 | 7.01 | 53 | 89 |

Ключевые сравнения:

- Bayes требует в среднем на `26.86%` меньше ходов, чем случайная стратегия.

То есть даже относительно хорошей практической эвристики байесовский выбор
остаётся лучшим.

## 6. Сравнение с внешней статистикой

С открытой статистикой есть проблема: большинство доступных публикаций и
гайдов используют **стандартный англоязычный флот** `5, 4, 3, 3, 2`, а не
русский вариант `4, 3, 3, 2, 2, 2, 1, 1, 1, 1`.

Поэтому я использую внешние числа только как **качественный ориентир**, а не как
прямое совпадение по абсолютным значениям.

### Что удалось найти

1. В статье Chanbin Park про probability density strategy сообщается, что против
   случайной расстановки кораблей probability density strategy в стандартном
   варианте давала среднее около `46.1` хода.
2. Публичный probability-targeting guide Rare Pike пишет, что полные
   probability-based алгоритмы обычно дают примерно `42–45` выстрелов, а
   человеческая игра чаще лежит в районе `60–75` выстрелов.

### Как это соотносится с нашим проектом

Наши числа:

- Bayes: `55.21`
- Random: `75.49`

По абсолютному значению наш Bayes хуже, чем `42–46` из внешних материалов, но
это не удивительно по двум причинам:

1. У нас **другой флот** и большее число кораблей, особенно однопалубных.
2. В коде реализована **локально байес-оптимальная** стратегия по occurrence
   matrix, а не полное решение глобальной POMDP-задачи.

При этом качественная картина совпадает:

- random заметно хуже;
- probability / Bayes-подход заметно лучше random.

## 7. Вывод об оптимальности

Если понимать «байес-оптимальность» как выбор следующего хода с максимальной
апостериорной вероятностью попадания, то реализованная стратегия действительно
оптимальна **в одношаговом байесовском смысле**.

По результатам симуляций:

- она уверенно выигрывает у случайной стратегии;
- распределение ходов у неё сдвинуто влево, то есть она заканчивает партии
  быстрее.

Поэтому вывод такой:

**байесовская стратегия в нашей постановке является лучшей из протестированных
стратегий и практически подтверждает свою эффективность, хотя строгая глобальная
оптимальность для всей игры целиком здесь не доказывается.**
